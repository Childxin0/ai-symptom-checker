# 风险控制

## 设计原则

1. **规则优先于模型**：规则引擎给出的风险等级与模型输出取 **较高** 等级（安全优先）。
2. **提示词层**：`prompts/risk-control.txt` 约束模型在 JSON 中的自我声明与禁用措辞。
3. **服务端层**：`validate.js` 检测建议中的高风险措辞（如暗示不必就医），触发 **安全改写**。
4. **降级层**：解析失败或模型异常时在服务端合并保守话术；进程级异常由统一错误封装返回可渲染的保守响应（`success: false`）。

## 风险等级定义（简版）

| 等级 | 典型触发（示例） |
| --- | --- |
| HIGH | 胸痛、呼吸困难、意识障碍、自杀意念、大量出血等关键词 |
| MEDIUM | 持续高热、严重呕吐、便血、晕厥、心悸相关表述等 |
| LOW | 未命中上述关键词时的默认层级（仍提示关注变化） |

关键词维护位置：`risk-engine/keywords.js`。

## Explainability

面向用户的叙述由 `risk-engine/explainNL.js` 的 `buildNaturalExplainability` 生成（不出现内部术语）；对外通过 `insight.narrative` 字段呈现。

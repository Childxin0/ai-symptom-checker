## 解决的核心痛点

### 传统医疗Chatbot的问题

1. **误用风险**：任何输入都返回医疗建议，可能被误用为通用聊天工具
2. **误判风险**：信息不足时盲目判低风险，可能漏诊高危症状
3. **LLM幻觉**：过度依赖大模型，急症识别不可靠
4. **黑盒输出**：无法解释判断依据，用户和医生难以信任
5. **系统脆弱**：LLM不可用时整个系统崩溃

### 我们的解决方案

1. **输入意图识别**：三状态过滤（非医疗、信息不足、有效症状），避免误用
2. **规则优先分层**：70+ Red Flag规则保底，急症识别100%可靠
3. **追问机制**：信息不足时标准化追问，而非盲目判低风险
4. **Fallback降级**：LLM失败时，规则+关键词仍能提供基本服务
5. **可解释性**：显示触发规则和AI推理过程，增强信任

## 核心功能模块

### 1. 输入意图识别（Input Validation）

**实现方式**：
- 50+非医疗关键词库（技术、职业、学习等）
- 40+症状关键词库（疼痛、发热、咳嗽等）
- 15+紧急症状关键词（车祸、大出血、昏迷等）

**产品价值**：
- 过滤非医疗输入（如"帮我写简历"），避免系统误用
- 识别信息不足（如"不舒服"），触发追问而非误判低风险

**测试结果**：6/6通过（非医疗3个，信息不足3个）

---

### 2. 症状结构化（Symptom Structuring）

**技术方案**：
- LLM：Claude Sonnet 4.5（温度0.1）
- Prompt工程：明确要求完整症状、时间、体温提取
- Pydantic V2：严格数据验证
- Fallback：关键词匹配 + 正则提取

**示例**：
```
输入："头疼三天，昨晚发烧38.5度，还有点恶心"
输出：
  symptoms: ["头疼", "发烧", "恶心"]
  duration: "三天"  ✅
  temperature: 38.5°C  ✅
  severity: "中度"
```

**测试结果**：症状识别100%，duration提取100%

---

### 3. Red Flag风险分层（Risk Layering）

**设计理念**：**规则优先（Rules-First）+ LLM辅助**

**规则体系**：
- 70+条Red Flag规则
- 9大高危类别：外伤、呼吸、神经、急腹症、心血管、感染、心理危机、中毒、儿科/产科
- 4级风险：EMERGENCY/HIGH/MEDIUM/LOW

**核心逻辑**：
1. 规则引擎优先评估（确定性强）
2. LLM辅助结构化（灵活性强）
3. 规则触发 → 直接判定高风险，不依赖LLM
4. 多信号叠加 → 动态评分

**示例**：
- "车祸大出血" → 触发规则 → EMERGENCY/100 → 立即拨打120
- "发烧头疼恶心" → 规则评分 → MEDIUM/64 → 24-48小时就医
- "轻微感冒" → 规则评分 → LOW/20 → 观察3天

**测试结果**：3/3准确，分层合理

---

### 4. Follow-up追问机制

**触发条件**：`input_type = insufficient_symptom`

**标准问题**（4个）：
1. 具体不适部位
2. 持续时间
3. 严重程度
4. 伴随症状

**产品设计**：
- 不显示风险卡（避免误判）
- 蓝色信息卡引导补充
- 用户补充后重新提交分析

**测试结果**：3/3触发追问，无误判低风险

---

### 5. Fallback降级机制

**三层降级**：
1. LLM结构化失败 → 关键词匹配 + 正则提取
2. 使用规则引擎评估（Rules-only模式）
3. 全部失败 → 保守评估MEDIUM（不低估风险）

**关键设计**：
- 高危识别保底：即使LLM失败，规则引擎仍能识别"大出血"等
- 保守策略：失败时默认MEDIUM而非LOW
- 安全优先：不因系统故障低估用户风险

**测试验证**：关闭LLM仍能识别EMERGENCY ✅

---

### 6. Explainability可解释性

**展示内容**：
- 触发的规则ID和中文解释
- AI分析推理过程
- 风险判断依据

**产品价值**：
- 用户信任：理解系统判断逻辑
- 医生审核：便于专家review
- 系统调试：快速定位问题
